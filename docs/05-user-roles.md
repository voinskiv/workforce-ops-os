# User Roles

## Management

Needs:

- operational overview
- risk signals
- profitability direction
- open decisions
- partner quality
- customer risk
- no-show trends
- high-priority cases

Actions:

- view dashboard
- review critical cases
- review project/customer status
- approve sensitive actions later

## Disposition

Needs:

- employee status
- assignments
- planning restrictions
- shifts
- customer/site needs
- no-shows
- replacements

Actions:

- create assignments
- update assignment status
- flag conflicts
- create replacement cases
- update employee status

## Backoffice

Needs:

- time issues
- customer rules
- operational cases
- document statuses
- reimbursements/fines as cases
- billing-adjacent checks

Actions:

- create time entries
- create time issues
- update correction status
- create cases
- update document status
- prepare data for billing

## Foreman

Needs:

- today's planned employees
- new employees arriving
- no-show reporting
- time issue confirmation
- performance or training notes

Actions:

- confirm appeared / no-show
- confirm first day
- comment on time issue
- create operational case
- mark worker as trained or needs training

## Partner Coordinator

Needs:

- list of expected workers
- no-show/replacement requests
- missing document requests
- partner-related cases

Actions:

- provide replacement info
- update worker status through internal staff or future portal
- respond to cases

## System Admin

Needs:

- users
- roles
- configuration
- seed data
- customer rules
- system settings

Actions:

- manage users
- manage master data
- configure customer/site rules
